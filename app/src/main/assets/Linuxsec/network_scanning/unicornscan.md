# unicornscan 命令

## 命令说明
unicornscan是一款功能强大的网络扫描工具，专注于高性能的TCP和UDP扫描。它采用异步网络I/O技术，可以同时扫描多个目标和端口，是网络安全评估和端口发现的重要工具。

## 语法
```
unicornscan [选项] 目标
```

## 常用选项
- `-i <接口>`：指定网络接口
- `-m <模式>`：设置扫描模式，如 tcp、udp、icmp
- `-p <端口>`：指定端口范围，如 1-1000 或 80,443
- `-r <速率>`：设置扫描速率（数据包/秒）
- `-c <数量>`：设置并发连接数
- `-f <文件>`：从文件读取目标列表
- `-o <文件>`：输出结果到文件
- `-v`：详细模式
- `-h`：显示帮助信息
- `--version`：显示版本信息

## 技术原理

unicornscan采用以下技术实现高性能扫描：

1. 异步网络I/O：使用libpcap和libnet库实现异步网络操作
2. 多线程并发：使用多个线程同时处理扫描任务
3. 智能速率控制：根据网络条件自动调整扫描速率
4. 无状态扫描：不维护连接状态，提高扫描效率
5. 分布式扫描：支持多机协同扫描

unicornscan的扫描速度可以达到每秒数万个数据包，比传统的扫描工具快很多。

## 示例

### 基本扫描
```bash
# TCP端口扫描
unicornscan -m tcp 192.168.1.0/24:80

# UDP端口扫描
unicornscan -m udp 192.168.1.0/24:53

# 扫描多个端口
unicornscan -m tcp 192.168.1.0/24:80,443,22

# 扫描端口范围
unicornscan -m tcp 192.168.1.0/24:1-1000
```

### 高级扫描
```bash
# 设置扫描速率为每秒1000个数据包
unicornscan -r 1000 -m tcp 192.168.1.0/24:80

# 设置并发连接数为100
unicornscan -c 100 -m tcp 192.168.1.0/24:80

# 详细模式扫描
unicornscan -v -m tcp 192.168.1.0/24:80

# 从文件读取目标列表
unicornscan -f targets.txt -m tcp :80
```

### 输出选项
```bash
# 输出结果到文件
unicornscan -o results.txt -m tcp 192.168.1.0/24:80

# 同时输出到终端和文件
unicornscan -o results.txt -v -m tcp 192.168.1.0/24:80
```

### 网络设置
```bash
# 指定网络接口
unicornscan -i eth0 -m tcp 192.168.1.0/24:80

# 扫描多个目标
unicornscan -m tcp 192.168.1.1-100:80

# 扫描整个网段
unicornscan -m tcp 192.168.1.0/24:80
```

## 高级技巧

### 与其他工具结合
```
# 扫描结果导入nmap进行详细扫描
unicornscan -o results.txt -m tcp 192.168.1.0/24:80 && grep "open" results.txt | cut -d " " -f 1 | nmap -iL -

# 扫描并过滤结果
unicornscan -m tcp 192.168.1.0/24:80 | grep "open"

# 批量扫描多个端口
for port in 80 443 22 21 3389; do
  unicornscan -m tcp 192.168.1.0/24:$port -o scan_$port.txt
done
```

### 性能优化
- 根据网络带宽调整扫描速率
- 对于大型网络，使用多个unicornscan实例
- 合理设置并发连接数
- 使用 `-r` 选项控制扫描速率

### 安全注意事项
- 获得授权后再进行扫描
- 注意扫描速率，避免对网络造成影响
- 遵守相关法律法规
- 避免扫描互联网上的随机IP地址

### 常见问题解决

- **问题**：扫描速度慢
  **解决方法**：增加扫描速率，使用更强大的硬件

- **问题**：扫描结果不准确
  **解决方法**：降低扫描速率，增加并发连接数

- **问题**：被防火墙阻止
  **解决方法**：调整扫描速率，使用不同的扫描模式

- **问题**：内存使用过高
  **解决方法**：减少扫描范围，使用更小的并发连接数
