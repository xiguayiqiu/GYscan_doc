# nmap 命令

## 命令说明
Nmap（Network Mapper）是一款功能强大的网络扫描和安全评估工具，用于发现网络上的主机、开放端口、运行的服务及其版本信息，以及操作系统类型等。它是网络安全专业人员的必备工具，广泛应用于网络安全评估、漏洞扫描和网络管理等领域。

## 语法
```
nmap [选项] [目标]
```

## 常用选项

### 扫描类型
- `-sS`：TCP SYN扫描（半开放扫描），不完成TCP三次握手，速度快且隐蔽
- `-sT`：TCP连接扫描，完成完整的TCP三次握手
- `-sU`：UDP扫描，用于探测UDP端口
- `-sA`：TCP ACK扫描，用于探测防火墙规则
- `-sW`：TCP窗口扫描，通过分析TCP窗口大小来判断端口状态
- `-sM`：TCP Maimon扫描，利用FIN/ACK包探测端口
- `-sN`、`-sF`、`-sX`：TCP Null、FIN、Xmas扫描，用于绕过某些防火墙

### 服务和版本检测
- `-sV`：探测开放端口以确定服务/版本信息
- `--version-intensity <level>`：设置版本探测强度（0-9）
- `--version-light`：使用轻量级版本探测
- `--version-all`：尝试所有版本探测方法

### 操作系统检测
- `-O`：启用操作系统检测
- `--osscan-limit`：限制操作系统检测范围
- `--osscan-guess`：尝试猜测操作系统类型

### 脚本扫描
- `-sC`：使用默认脚本集合
- `--script <script>`：指定要运行的脚本
- `--script-args <args>`：为脚本提供参数
- `--script-trace`：显示脚本执行过程

### 扫描优化
- `-T<0-5>`：设置时序模板（0-5，数字越大速度越快）
  - `T0`：偏执模式，非常慢但隐蔽
  - `T1`： Sneaky模式
  - `T2`： Polite模式
  - `T3`： Normal模式（默认）
  - `T4`： Aggressive模式
  - `T5`： Insane模式，非常快但可能导致网络拥塞
- `-F`：快速扫描，只扫描常用端口
- `--top-ports <n>`：只扫描最常用的n个端口
- `--min-rate <rate>`：设置最小发包速率
- `--max-rate <rate>`：设置最大发包速率

### 目标指定
- `-iL <file>`：从文件中读取目标列表
- `-iR <num>`：随机选择num个目标
- `--exclude <host>`：排除指定主机
- `--exclude-file <file>`：从文件中读取要排除的主机

### 输出格式
- `-oN <file>`：输出正常格式到文件
- `-oX <file>`：输出XML格式到文件
- `-oG <file>`：输出grepable格式到文件
- `-oA <basename>`：输出所有格式到文件

### 其他选项
- `-A`：启用操作系统检测、版本检测、脚本扫描和跟踪路由
- `-p <port>`：指定端口范围（如 1-1000,80,443）
- `-6`：启用IPv6扫描
- `-Pn`：跳过主机发现，直接扫描端口
- `-PS`：TCP SYN Ping
- `-PA`：TCP ACK Ping
- `-PU`：UDP Ping
- `-PR`：ARP Ping（适用于局域网）

## 扫描技术详解

### TCP SYN扫描（-sS）
这是Nmap的默认扫描方式，也称为半开放扫描。它发送SYN包到目标端口，然后根据响应判断端口状态：
- 如果收到SYN/ACK响应，则端口开放
- 如果收到RST响应，则端口关闭
- 如果没有响应，则端口被过滤

这种扫描方式的优点是速度快、隐蔽性好，因为它不完成TCP三次握手，减少了被防火墙检测的可能性。

### UDP扫描（-sU）
UDP扫描用于探测UDP端口的开放状态。由于UDP是无连接协议，扫描速度较慢，且结果可能不够准确。

### 操作系统检测（-O）
Nmap通过分析TCP/IP栈的特征来识别操作系统类型和版本。这对于漏洞评估和安全加固非常重要。

### 脚本扫描（-sC）
Nmap内置了大量脚本，用于执行各种高级扫描任务，如漏洞检测、服务枚举、安全检查等。

## 示例

### 基本扫描
```bash
# 扫描单个IP
nmap 192.168.1.1

# 扫描多个IP
nmap 192.168.1.1 192.168.1.2 192.168.1.3

# 扫描整个网段
nmap 192.168.1.0/24
```

### 服务和版本检测
```bash
# 探测服务和版本
nmap -sV 192.168.1.1

# 启用操作系统检测和脚本扫描
nmap -A 192.168.1.1
```

### 端口扫描
```bash
# 扫描指定端口
nmap -p 80,443,22 192.168.1.1

# 扫描端口范围
nmap -p 1-1000 192.168.1.1

# 快速扫描常用端口
nmap -F 192.168.1.1

# 扫描前100个最常用端口
nmap --top-ports 100 192.168.1.1
```

### 高级扫描
```bash
# 半开放扫描+服务检测+操作系统检测
nmap -sS -sV -O 192.168.1.1

# 完整扫描（推荐用于安全评估）
nmap -A -T4 192.168.1.0/24

# UDP扫描
nmap -sU -p 53,161,162 192.168.1.1

# 防火墙规则探测
nmap -sA 192.168.1.1
```

### 输出格式
```bash
# 输出到文件（正常格式）
nmap -oN scan.txt 192.168.1.0/24

# 输出到文件（XML格式）
nmap -oX scan.xml 192.168.1.0/24

# 输出所有格式
nmap -oA scan 192.168.1.0/24
```

### 特殊场景扫描
```bash
# 绕过防火墙检测
nmap -sS -sA -sF -sX -T4 192.168.1.1

# 扫描IPv6地址
nmap -6 2001:db8::1

# 扫描内网（使用ARP扫描提高速度）
nmap -PR 192.168.1.0/24
```

## 适用场景

### 网络安全评估
- 识别网络中的主机和服务
- 发现潜在的安全漏洞
- 评估网络的安全状况
- 生成安全报告

### 漏洞扫描
- 检测系统和服务的已知漏洞
- 评估漏洞的严重程度
- 提供漏洞修复建议

### 网络拓扑发现
- 绘制网络拓扑图
- 识别网络设备和连接
- 发现网络中的异常设备

### 服务管理
- 监控网络服务的运行状态
- 检测未授权的服务
- 确保服务的安全配置

###  incident响应
- 分析网络入侵事件
- 识别攻击源和攻击路径
- 评估攻击的影响范围

## 高级技巧

### 使用Nmap脚本引擎
Nmap脚本引擎（NSE）是Nmap的强大功能之一，它允许用户编写和执行自定义脚本。常用的NSE脚本类别包括：

- `auth`：身份验证相关脚本
- `broadcast`：广播相关脚本
- `brute`：暴力破解脚本
- `default`：默认脚本集合
- `discovery`：网络发现脚本
- `dos`：拒绝服务测试脚本
- `exploit`：漏洞利用脚本
- `external`：外部服务脚本
- `fuzzer`：模糊测试脚本
- `intrusive`：侵入性脚本
- `malware`：恶意软件检测脚本
- `safe`：安全脚本
- `version`：版本检测脚本
- `vuln`：漏洞检测脚本

### 示例：使用NSE脚本
```bash
# 检测常见漏洞
nmap --script vuln 192.168.1.1

# 暴力破解SSH密码
nmap --script ssh-brute 192.168.1.1

# 检测Web应用漏洞
nmap --script http-vuln* 192.168.1.1

# 发现网络共享
nmap --script smb-enum-shares 192.168.1.1
```

### 性能优化
- 使用 `-T4` 或 `-T5` 提高扫描速度（但可能增加被检测的风险）
- 使用 `-F` 或 `--top-ports` 减少扫描的端口数量
- 使用 `-oA` 同时输出多种格式，方便后续分析
- 对于大型网络，使用 `-iL` 从文件读取目标列表

### 安全注意事项
- 获得授权后再进行扫描，避免触犯法律
- 避免在生产网络上使用过于激进的扫描选项
- 注意扫描可能对目标系统造成的影响
- 合理使用扫描速度，避免网络拥塞

## 总结
Nmap是一款功能强大、灵活的网络扫描工具，提供了丰富的选项和功能，适用于各种网络安全场景。通过掌握Nmap的使用方法，安全专业人员可以更有效地评估网络安全状况，发现潜在的安全问题，并采取相应的防护措施。

- 内置强大的脚本引擎
- 跨平台支持
- 详细的文档和活跃的社区

作为网络安全工具的瑞士军刀，Nmap是每个安全专业人员必备的工具之一。
