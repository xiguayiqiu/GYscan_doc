# zmap 命令

## 命令说明
zmap是一款功能强大的高速网络扫描工具，用于快速扫描互联网范围内的IP地址和端口。它采用无状态扫描技术，可以在短时间内扫描大量IP地址，是大规模网络安全评估和端口发现的重要工具。

## 语法
```
zmap [选项] [目标]
```

## 常用选项
- `-p <端口>`：指定端口，如 80 或 80,443
- `-n <数量>`：设置扫描的IP数量
- `-B <带宽>`：设置带宽限制，如 10M
- `-o <文件>`：输出结果到文件
- `-O <格式>`：设置输出格式，如 json、csv、grepable
- `-i <接口>`：指定网络接口
- `-S <IP>`：设置源IP地址
- `-s <端口>`：设置源端口
- `--rate <速率>`：设置数据包发送速率
- `--max-targets <数量>`：设置最大目标数量
- `--probes <数量>`：设置每个目标的探测次数
- `--timeout <秒>`：设置超时时间
- `--help`：显示帮助信息
- `--version`：显示版本信息

## 技术原理

zmap采用以下技术实现高速扫描：

1. 无状态扫描：不维护连接状态，只发送数据包并等待响应
2. 并行处理：使用多线程并行发送和接收数据包
3. 优化的网络栈：绕过操作系统的网络栈，直接使用原始套接字
4. 智能速率控制：根据网络条件自动调整发送速率
5. 分布式扫描：支持多机协同扫描

zmap的扫描速度可以达到每秒数百万个数据包，比传统的扫描工具快几个数量级。

## 示例

### 基本扫描
```bash
# 扫描整个互联网的80端口
zmap -p 80

# 扫描指定网段的443端口
zmap -p 443 192.168.1.0/24

# 扫描多个端口
zmap -p 80,443,22

# 限制扫描的IP数量
zmap -p 80 -n 10000
```

### 高级扫描
```bash
# 设置带宽限制为10Mbps
zmap -p 80 -B 10M

# 设置数据包发送速率为每秒10000个
zmap -p 80 --rate 10000

# 设置每个目标的探测次数为3
zmap -p 80 --probes 3

# 设置超时时间为5秒
zmap -p 80 --timeout 5
```

### 输出格式
```bash
# 输出为JSON格式
zmap -p 80 -O json -o results.json

# 输出为CSV格式
zmap -p 80 -O csv -o results.csv

# 输出为grepable格式
zmap -p 80 -O grepable -o results.txt
```

### 网络设置
```bash
# 指定网络接口
zmap -p 80 -i eth0

# 设置源IP地址
zmap -p 80 -S 192.168.1.100

# 设置源端口
zmap -p 80 -s 12345

# 最大目标数量
zmap -p 80 --max-targets 1000000
```

## 高级技巧

### 与其他工具结合
```bash
# 扫描结果导入nmap进行详细扫描
zmap -p 80 -o - | nmap -iL -

# 扫描并过滤结果
zmap -p 80 -O json | jq '.[] | select(.success == true)'

# 批量扫描多个端口
for port in 80 443 22 21 3389; do
  zmap -p $port -o results_$port.txt

done
```

### 性能优化
- 根据网络带宽调整扫描速率
- 对于大型扫描，使用多个zmap实例
- 合理设置数据包大小和TTL值
- 使用SSD存储提高输出性能

### 安全注意事项
- 获得授权后再进行扫描
- 注意扫描速率，避免对网络造成影响
- 遵守相关法律法规
- 避免扫描互联网上的随机IP地址

### 常见问题解决

- **问题**：扫描速度慢
  **解决方法**：增加扫描速率，使用更强大的硬件

- **问题**：扫描结果不准确
  **解决方法**：降低扫描速率，增加探测次数

- **问题**：被防火墙阻止
  **解决方法**：调整扫描速率，使用不同的源端口

- **问题**：内存使用过高
  **解决方法**：减少扫描范围，使用更小的批处理大小
