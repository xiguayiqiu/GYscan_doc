# sed 命令

## 命令说明
sed（Stream Editor）是一款功能强大的流编辑器，用于对文本进行替换、删除、插入、追加等操作。它可以逐行处理文本流，是文本处理和自动化脚本编写的重要工具。

## 语法
```
sed [选项] '命令' 文件
```

## 常用选项
- `-i`：直接修改文件，不创建备份
- `-i.bak`：直接修改文件，创建备份文件
- `-e`：执行多个命令
- `-n`：只显示匹配的行
- `-f`：从文件读取命令
- `-r`：使用扩展正则表达式
- `-E`：与 `-r` 相同，使用扩展正则表达式
- `-s`：将多个文件视为独立的流，而不是单个连续流

## 基本命令

### 替换命令
- `s/模式/替换/[标志]`：替换匹配的文本
  - `g`：全局替换
  - `i`：忽略大小写
  - `p`：打印替换后的行
  - `w 文件`：将替换后的行写入文件
  - `数字`：只替换第n次出现的匹配

### 删除命令
- `d`：删除匹配的行

### 打印命令
- `p`：打印匹配的行
- `n`：读取下一行，跳过当前行的处理

### 插入和追加命令
- `i\文本`：在匹配行前插入文本
- `a\文本`：在匹配行后追加文本
- `c\文本`：替换匹配的行

### 其他命令
- `h`：将模式空间复制到保持空间
- `H`：将模式空间追加到保持空间
- `g`：将保持空间复制到模式空间
- `G`：将保持空间追加到模式空间
- `x`：交换模式空间和保持空间
- `y/源字符/目标字符/`：字符转换

## 地址范围

sed可以指定操作的地址范围：
- `n`：第n行
- `n,m`：第n到m行
- `/模式/`：匹配模式的行
- `/模式1/,/模式2/`：从匹配模式1的行到匹配模式2的行
- `n,/模式/`：从第n行到匹配模式的行
- `/模式/,m`：从匹配模式的行到第m行

## 正则表达式

### 基本元字符
- `.`：匹配任意单个字符
- `*`：匹配前面的字符零次或多次
- `^`：匹配行首
- `$`：匹配行尾
- `[]`：匹配方括号内的任意一个字符
- `[^]`：匹配不在方括号内的任意一个字符
- `\`：转义特殊字符

### 扩展正则表达式（需要 `-r` 或 `-E` 选项）
- `+`：匹配前面的字符一次或多次
- `?`：匹配前面的字符零次或一次
- `()`：分组
- `|`：或操作
- `{n}`：恰好n次
- `{n,}`：至少n次
- `{n,m}`：至少n次，最多m次

## 示例

### 基本替换
```bash
# 替换第一个匹配的old为new
sed 's/old/new/' file.txt

# 全局替换所有old为new
sed 's/old/new/g' file.txt

# 替换第2次出现的old为new
sed 's/old/new/2' file.txt

# 忽略大小写替换
sed 's/old/new/gi' file.txt

# 直接修改文件
sed -i 's/old/new/g' file.txt

# 直接修改文件并创建备份
sed -i.bak 's/old/new/g' file.txt
```

### 删除操作
```bash
# 删除第5行
sed '5d' file.txt

# 删除第2到5行
sed '2,5d' file.txt

# 删除包含error的行
sed '/error/d' file.txt

# 删除以#开头的行
sed '/^#/d' file.txt

# 删除空行
sed '/^$/d' file.txt
```

### 打印操作
```bash
# 打印包含error的行
sed -n '/error/p' file.txt

# 打印第3到7行
sed -n '3,7p' file.txt

# 打印从包含start的行到包含end的行
sed -n '/start/,/end/p' file.txt
```

### 插入和追加
```bash
# 在第3行前插入文本
sed '3i\插入的文本' file.txt

# 在第3行后追加文本
sed '3a\追加的文本' file.txt

# 在包含pattern的行前插入文本
sed '/pattern/i\插入的文本' file.txt

# 在包含pattern的行后追加文本
sed '/pattern/a\追加的文本' file.txt

# 替换包含pattern的行
sed '/pattern/c\替换的文本' file.txt
```

### 高级替换
```bash
# 使用捕获组进行替换
sed 's/\(.*\)\@\(.*\)/\2@\1/' file.txt

# 使用扩展正则表达式
sed -r 's/(.*)@(.*)/\2@\1/' file.txt

# 替换多个模式
sed -e 's/old1/new1/g' -e 's/old2/new2/g' file.txt

# 从文件读取命令
sed -f commands.sed file.txt
```

### 与其他命令结合
```bash
# 过滤并替换文本
grep 'error' /var/log/syslog | sed 's/error/ERROR/g'

# 处理管道输入
ls -l | sed 's/^.*\ ///'

# 批量重命名文件
ls *.txt | sed 's/\.txt$/.md/' | xargs -I {} mv {}.txt {}

# 清理日志文件
sed -i '/^$/d' /var/log/syslog
```

## 高级技巧

### 使用保持空间
```bash
# 保存和恢复模式空间
sed '1h;2,$H;$g;$p' file.txt

# 反转文件内容
sed '1!G;h;$!d' file.txt

# 去重（相邻行）
sed '$!N;/^\(.*\)\n\1$/!P;D'
```

### 多行处理
```bash
# 合并多行
sed 'N;s/\n/ /' file.txt

# 处理跨行模式
sed '/^start/,/^end/{/^start/{h;d};/^end/{H;x;s/\n/ /;p;d};H}' file.txt
```

### 复杂替换
```bash
# 替换HTML标签
sed 's/<[^>]*>//g' file.html

# 替换URL中的参数
sed 's/\?.*//' file.txt

# 格式化日期
sed 's/\([0-9]\{4\}\)-\([0-9]\{2\}\)-\([0-9]\{2\}\)/\3-\2-\1/' file.txt
```

### 性能优化
- 对于大型文件，使用更具体的地址范围
- 避免使用复杂的正则表达式
- 对于简单替换，考虑使用 `tr` 命令
- 使用 `-i` 选项时要小心，建议先备份

### 常见问题解决
- **问题**：替换包含特殊字符的模式
  **解决方法**：使用 `\` 转义特殊字符，或使用不同的分隔符

- **问题**：直接修改文件时出错
  **解决方法**：使用 `-i.bak` 创建备份，或先测试命令

- **问题**：处理多行文本
  **解决方法**：使用 `N`、`H`、`g` 等命令进行多行处理
