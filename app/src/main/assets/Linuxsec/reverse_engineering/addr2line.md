# addr2line 命令

## 命令说明
addr2line是一款功能强大的地址转换工具，用于将程序地址转换为对应的源代码文件名和行号。它可以帮助开发者和逆向工程师定位程序崩溃的位置，分析程序的执行流程，是调试和逆向工程的重要工具。

## 语法
```
addr2line [选项] 地址 [地址...]
```

## 常用选项
- `-e <文件>`：指定可执行文件或共享库
- `-f`：显示函数名
- `-C`：demangle C++函数名
- `-i`：处理内联函数
- `-j <段>`：指定段名
- `--help`：显示帮助信息
- `--version`：显示版本信息

## 技术原理

addr2line通过以下步骤工作：

1. 读取可执行文件或共享库中的调试信息
2. 根据指定的地址查找对应的源代码位置
3. 显示源代码文件名和行号

addr2line需要可执行文件或共享库中包含调试信息（通常使用 `-g` 选项编译）才能正常工作。

## 示例

### 基本操作
```bash
# 将地址转换为源代码位置
addr2line -e a.out 0x400567

# 显示函数名
addr2line -f -e a.out 0x400567

# demangle C++函数名
addr2line -f -C -e a.out 0x400567

# 处理内联函数
addr2line -f -i -e a.out 0x400567
```

### 多个地址
```bash
# 转换多个地址
addr2line -e a.out 0x400567 0x400678 0x400789

# 显示多个地址的函数名和源代码位置
addr2line -f -e a.out 0x400567 0x400678 0x400789
```

### 共享库
```bash
# 转换共享库中的地址
addr2line -e libexample.so 0x12345

# 显示共享库中地址的函数名
addr2line -f -e libexample.so 0x12345
```

### 实用场景
```bash
# 分析崩溃日志
# 假设崩溃日志显示程序在地址 0x400567 崩溃
addr2line -f -e a.out 0x400567

# 分析核心转储文件
# 假设核心转储文件显示程序在地址 0x400567 崩溃
gdb -c core -e a.out -ex "info line *0x400567" -ex quit

# 或者使用 addr2line
addr2line -f -e a.out 0x400567
```

## 高级技巧

### 与其他工具结合
```bash
# 结合gdb分析崩溃
# 1. 运行程序并获取崩溃地址
gdb -ex "run" -ex "info line *$pc" -ex quit a.out

# 2. 使用addr2line转换地址
addr2line -f -e a.out 0x400567

# 结合objdump分析代码
addr2line -f -e a.out 0x400567 && objdump -d -j .text a.out | grep -A 10 0x400567

# 分析核心转储文件中的调用栈
addr2line -f -e a.out $(grep -E "^#\d+" backtrace.txt | awk '{print $2}')
```

### 批量处理
```bash
# 批量转换多个地址
cat addresses.txt | xargs addr2line -f -e a.out

# 分析崩溃日志中的所有地址
grep -E "0x[0-9a-f]+" crash.log | awk '{print $NF}' | xargs addr2line -f -e a.out
```

### 性能优化
- 对于频繁的操作，创建脚本自动化处理
- 使用 `-f` 选项同时显示函数名和源代码位置
- 对于C++程序，使用 `-C` 选项demangle函数名
- 对于内联函数，使用 `-i` 选项处理

### 常见问题解决

- **问题**：无法找到源代码位置
  **解决方法**：确保可执行文件或共享库中包含调试信息（使用 `-g` 选项编译）

- **问题**：函数名显示为乱码
  **解决方法**：对于C++程序，使用 `-C` 选项demangle函数名

- **问题**：地址不在任何段中
  **解决方法**：确保地址是有效的程序地址，检查可执行文件是否正确

- **问题**：无法打开文件
  **解决方法**：确保可执行文件或共享库存在，并且有读取权限
