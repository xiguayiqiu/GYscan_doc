# fsck 命令

## 命令说明

`fsck` 是 Linux 系统中用于检查和修复文件系统的命令，它可以检查文件系统的完整性并修复错误。

## 语法

```bash
fsck [选项] [设备]
```

## 常用选项

| 选项 | 描述 |
|------|------|
| `-a` | 自动修复错误，无需用户干预 |
| `-r` | 交互式修复错误，需要用户确认 |
| `-y` | 对所有问题回答 "yes" |
| `-n` | 只检查，不修复错误 |
| `-f` | 强制检查，即使文件系统被标记为干净 |
| `-t` | 指定文件系统类型 |

## 示例

### 检查并修复文件系统

```bash
fsck /dev/sda1
```

### 自动修复错误

```bash
fsck -a /dev/sda1
```

### 强制检查文件系统

```bash
fsck -f /dev/sda1
```

### 只检查，不修复错误

```bash
fsck -n /dev/sda1
```

## 注意事项

- `fsck` 命令需要 root 权限
- 检查文件系统时，文件系统应该处于未挂载状态
- 修复文件系统可能会导致数据丢失，建议在修复前备份重要数据
- 不同文件系统有专门的检查工具，如 `e2fsck` 用于 ext2/ext3/ext4 文件系统
