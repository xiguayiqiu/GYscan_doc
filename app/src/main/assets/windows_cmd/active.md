# active命令

## 命令简介

`active`命令是Windows磁盘管理工具中的一个命令，用于激活分区，使其成为活动分区。活动分区是系统启动时会被引导加载程序识别并启动的分区。

## 语法

```cmd
active
```

## 选项

`active`命令没有额外的选项参数。

## 示例

### 在磁盘管理工具中激活分区

```cmd
# 打开磁盘管理工具
diskpart

# 选择磁盘
select disk 0

# 选择分区
select partition 1

# 激活分区
active

# 退出磁盘管理工具
exit
```

## 高级用法

### 激活系统分区

```cmd
# 打开磁盘管理工具
diskpart

# 列出所有磁盘
list disk

# 选择包含系统分区的磁盘
select disk 0

# 列出磁盘上的所有分区
list partition

# 选择系统分区（通常是主分区）
select partition 1

# 激活分区
active

# 退出磁盘管理工具
exit
```

## 常见问题

### 问题：无法激活分区

**原因**：可能是分区类型不支持激活，或者磁盘是GPT格式。

**解决方法**：在GPT格式的磁盘上，不需要激活分区，而是通过EFI系统分区来引导。

### 问题：激活分区后系统无法启动

**原因**：可能是激活了错误的分区，或者分区中没有有效的操作系统。

**解决方法**：重新激活正确的系统分区，确保分区中包含有效的操作系统。

## 命令对比

| 命令 | 功能 | 特点 |
|------|------|------|
| `active` | 激活分区 | 使分区成为活动分区，用于MBR磁盘 |
| `select partition` | 选择分区 | 在diskpart中选择要操作的分区 |
| `list partition` | 列出分区 | 显示磁盘上的所有分区 |

## 注意事项

1. **权限要求**：使用diskpart工具需要管理员权限。

2. **磁盘格式**：`active`命令仅适用于MBR格式的磁盘，在GPT格式的磁盘上不需要激活分区。

3. **系统影响**：激活错误的分区可能导致系统无法启动，操作前请仔细确认。

4. **分区类型**：只有主分区可以被激活，扩展分区和逻辑分区不能被激活。

5. **适用场景**：`active`命令适用于在MBR磁盘上设置系统启动分区的场景。
