# skill - 发送信号给进程

## 1. 命令简介
skill 命令用于向进程发送信号，是 Linux 系统中常用的进程管理命令之一。它可以根据进程名、进程 ID 或其他属性向进程发送各种信号。

## 2. 命令语法
```bash
skill [选项] [信号] 进程选择器
```

## 3. 常用选项
| 选项 | 说明 |
|------|------|
| -f | 强制发送信号 |
| -i | 交互式操作，发送信号前询问 |
| -v | 显示详细信息 |
| -w | 等待进程终止 |
| -u | 按用户名选择进程 |
| -p | 按进程 ID 选择进程 |
| -t | 按终端选择进程 |
| -c | 按命令名选择进程 |

## 4. 常用信号
| 信号 | 数值 | 说明 |
|------|------|------|
| SIGHUP | 1 | 挂起信号，通常导致进程重新加载配置 |
| SIGINT | 2 | 中断信号（Ctrl+C），通常导致进程终止 |
| SIGQUIT | 3 | 退出信号（Ctrl+\），通常导致进程终止并生成核心转储 |
| SIGKILL | 9 | 强制终止信号，进程无法捕获，立即终止 |
| SIGTERM | 15 | 终止信号（默认），进程可以捕获并进行清理 |
| SIGSTOP | 17 | 停止信号，暂停进程执行 |
| SIGCONT | 18 | 继续信号，恢复暂停的进程 |
| SIGUSR1 | 10 | 用户定义信号 1 |
| SIGUSR2 | 12 | 用户定义信号 2 |

## 5. 实战示例

### 5.1 向进程发送终止信号
```bash
skill nginx
```

### 5.2 向进程发送强制终止信号
```bash
skill -9 nginx
```

### 5.3 交互式操作
```bash
skill -i nginx
```

### 5.4 显示详细信息
```bash
skill -v nginx
```

### 5.5 按用户名选择进程
```bash
skill -u username
```

### 5.6 按终端选择进程
```bash
skill -t pts/0
```

### 5.7 等待进程终止
```bash
skill -w nginx
```

### 5.8 强制发送信号
```bash
skill -f -9 nginx
```

## 6. 高级用法

### 6.1 批量发送信号
```bash
# 向多个进程发送信号
skill -9 process1 process2 process3

# 向所有进程发送信号（谨慎使用）
skill -9 -u username
```

### 6.2 结合正则表达式
```bash
# 向所有以 nginx 开头的进程发送信号
skill nginx*

# 向所有包含 http 的进程发送信号
skill *http*
```

### 6.3 与其他命令结合使用
```bash
# 先查看进程，再发送信号
ps aux | grep nginx
skill -9 nginx

# 结合 pgrep 命令
skill -9 $(pgrep nginx)
```

### 6.4 发送特定信号
```bash
# 发送 SIGHUP 信号（重新加载配置）
skill -1 nginx

# 发送 SIGSTOP 信号（暂停进程）
skill -17 nginx

# 发送 SIGCONT 信号（恢复进程）
skill -18 nginx
```

## 7. skill 与 pkill 的区别

| 特性 | skill | pkill |
|------|-------|-------|
| 语法 | 较旧，兼容性较差 | 较新，更灵活 |
| 选项 | 较少 | 较多 |
| 信号指定 | 作为选项 | 作为参数 |
| 进程选择 | 基于名称、用户等 | 基于名称、用户、终端等 |
| 可移植性 | 较差 | 较好 |
| 推荐使用 | 不推荐 | 推荐 |

## 8. 常见问题与解决方案

### 8.1 问题：skill: command not found
**解决方案**：在某些系统中，skill 命令可能不存在或已被 pkill 替代，使用 pkill 命令

### 8.2 问题：skill: no process found
**解决方案**：确保进程名称正确，或使用 ps 命令确认进程是否存在

### 8.3 问题：skill: permission denied
**解决方案**：确保有足够的权限，或使用 sudo 命令

### 8.4 问题：进程没有响应终止信号
**解决方案**：使用 -9 选项发送 SIGKILL 信号，这是强制终止信号

### 8.5 问题：skill 命令影响了不需要的进程
**解决方案**：使用更精确的进程选择器，或使用交互式模式（-i 选项）

## 9. 命令对比

| 命令 | 功能 | 适用场景 |
|------|------|----------|
| skill | 向进程发送信号 | 传统进程信号发送 |
| pkill | 向进程发送信号 | 现代进程信号发送，推荐使用 |
| kill | 向特定进程 ID 发送信号 | 精确发送信号给单个进程 |
| killall | 向所有同名进程发送信号 | 批量发送信号给同名进程 |
| pgrep | 查找进程 ID | 配合 kill 使用 |

## 10. 注意事项
- skill 命令根据进程名向进程发送信号
- skill 命令默认发送 SIGTERM 信号
- skill 命令在某些系统中可能已经被 pkill 命令替代
- 使用 SIGKILL 信号（-9）会强制终止进程，可能导致数据丢失
- 发送信号前，建议先使用 ps 命令确认进程信息
- 对于关键进程，建议使用 SIGTERM 信号（默认），让进程有机会进行清理
- 以 root 用户身份使用 skill 命令时要特别小心，避免误杀系统进程
