# vmstat命令

## 命令简介

`vmstat`命令用于显示虚拟内存状态，它可以提供有关系统内存、进程、CPU活动等信息的统计数据。

## 语法

```bash
vmstat [选项] [间隔时间] [次数]
```

## 选项

| 选项 | 描述 |
|------|------|
| `-a` | 显示活跃和非活跃内存 |
| `-f` | 显示启动以来的fork次数 |
| `-m` | 显示内存分配统计 |
| `-n` | 只在开始时显示头部信息 |
| `-s` | 显示内存和CPU的统计信息 |
| `-d` | 显示磁盘统计信息 |
| `-p <分区>` | 显示指定分区的统计信息 |
| `-S <单位>` | 使用指定的单位显示，如k、K、m、M |
| `-t` | 在输出中添加时间戳 |
| `-h`, `--help` | 显示帮助信息 |
| `-V`, `--version` | 显示版本信息 |

## 示例

### 显示系统虚拟内存状态

```bash
vmstat
```

### 每2秒显示一次，共显示5次

```bash
vmstat 2 5
```

### 显示活跃和非活跃内存

```bash
vmstat -a
```

### 显示内存分配统计

```bash
vmstat -m
```

### 显示磁盘统计信息

```bash
vmstat -d
```

### 显示指定分区的统计信息

```bash
vmstat -p /dev/sda1
```

### 显示内存和CPU的统计信息

```bash
vmstat -s
```

## 高级用法

### 监控系统状态

```bash
# 每1秒显示一次系统状态，按Ctrl+C停止
vmstat 1
```

### 结合其他命令使用

```bash
# 监控系统状态并保存到文件
vmstat 1 60 > system_monitor.txt

# 查看系统状态并过滤特定信息
vmstat -s | grep "memory"
```

## 常见问题

### 问题：vmstat显示的内存使用与free命令不符

**原因**：vmstat和free命令的计算方式略有不同，vmstat更关注虚拟内存的使用情况。

**解决方法**：两个命令结合使用，获得更全面的内存使用信息。

### 问题：vmstat显示的CPU使用率很高

**原因**：系统可能正在运行大量计算密集型任务。

**解决方法**：使用`top`或`htop`命令查看具体是哪些进程占用了CPU资源。

## 命令对比

| 命令 | 功能 | 特点 |
|------|------|------|
| `vmstat` | 显示虚拟内存状态 | 提供系统内存、进程、CPU活动等综合信息 |
| `free` | 显示内存使用情况 | 更专注于内存使用的详细信息 |
| `top` | 实时显示进程 | 实时监控进程的资源使用情况 |
| `iostat` | 显示I/O统计 | 更专注于磁盘I/O的统计信息 |

## 注意事项

1. **权限要求**：vmstat命令不需要特殊权限，普通用户也可以使用。

2. **输出解释**：vmstat的输出包含多个字段，需要了解每个字段的含义才能正确解读。

3. **持续监控**：使用间隔时间参数可以持续监控系统状态，适合排查性能问题。

4. **性能影响**：vmstat命令的执行对系统性能影响很小。

5. **适用场景**：vmstat适用于系统性能监控和故障排查，特别是内存和CPU相关的问题。
