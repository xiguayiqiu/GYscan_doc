# mpstat命令

## 命令简介

`mpstat`命令用于显示多处理器系统的CPU使用情况，它可以提供每个CPU核心的详细统计信息。

## 语法

```bash
mpstat [选项] [间隔时间] [次数]
```

## 选项

| 选项 | 描述 |
|------|------|
| `-P <CPU>` | 显示指定CPU的统计信息，`ALL`表示所有CPU |
| `-u` | 显示CPU使用率统计（默认） |
| `-I <选项>` | 显示中断统计，选项可以是`SUM`、`CPU`、`SCPU` |
| `-A` | 显示所有统计信息 |
| `-V` | 显示版本信息 |

## 示例

### 显示所有CPU的使用情况

```bash
mpstat
```

### 每2秒显示一次，共显示5次

```bash
mpstat 2 5
```

### 显示指定CPU的使用情况

```bash
# 显示CPU 0的使用情况
mpstat -P 0

# 显示所有CPU的使用情况
mpstat -P ALL
```

### 显示中断统计信息

```bash
mpstat -I SUM
```

### 显示所有统计信息

```bash
mpstat -A
```

## 高级用法

### 监控CPU使用情况

```bash
# 每1秒显示一次所有CPU的使用情况，按Ctrl+C停止
mpstat -P ALL 1
```

### 结合其他命令使用

```bash
# 监控CPU使用情况并保存到文件
mpstat -P ALL 1 60 > cpu_monitor.txt

# 查看CPU使用趋势
mpstat 2 10 | grep -A 5 "CPU"
```

## 常见问题

### 问题：mpstat命令不存在

**原因**：系统中没有安装sysstat包。

**解决方法**：安装sysstat包，例如在Ubuntu上执行`sudo apt install sysstat`。

### 问题：mpstat显示的CPU使用率很高

**原因**：系统可能正在运行大量计算密集型任务。

**解决方法**：使用`top`或`htop`命令查看具体是哪些进程占用了CPU资源。

## 命令对比

| 命令 | 功能 | 特点 |
|------|------|------|
| `mpstat` | 显示多处理器CPU使用情况 | 提供每个CPU核心的详细统计 |
| `top` | 实时显示进程 | 实时监控进程的资源使用情况 |
| `vmstat` | 显示虚拟内存状态 | 提供系统内存、进程、CPU活动等综合信息 |
| `iostat` | 显示I/O统计信息 | 提供CPU和磁盘I/O的详细统计 |

## 注意事项

1. **权限要求**：mpstat命令不需要特殊权限，普通用户也可以使用。

2. **安装要求**：mpstat命令属于sysstat包，需要单独安装。

3. **输出解释**：mpstat的输出包含多个字段，需要了解每个字段的含义才能正确解读。

4. **持续监控**：使用间隔时间参数可以持续监控CPU状态，适合排查性能问题。

5. **适用场景**：mpstat适用于多处理器系统的CPU性能监控和故障排查，特别是需要了解每个CPU核心使用情况的场景。
